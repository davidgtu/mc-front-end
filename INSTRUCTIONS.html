ManCrates Frontend Simulation

Before Starting

The goals of this exercise are to:

  • expose you to real problems that you will be expected to solve daily
  • follow instructions and build UIs with HTML and CSS
  • test your ability to write HTML and CSS according to our guidelines

You should approach this project as if it was going to go into production on a high-traffic website immediately after submission where millions of people will see your interfaces on many different devices. Indeed, we are interested in seeing what your standard is for a project like this as a professional engineer.

We expect that, excluding time taken to set up your system and read the documentation, you should be able to comfortably complete this exercise in roughly 3-4 hours. You should go as fast as you can without sacrificing quality in the submission. We would like to get a complete submission even if it takes a longer than expected or if you want to work on it in parts. To that end you have up to 3 days after receiving the project to finish. As a benchmark, our solution took less than 3 hours to complete in a single sitting and didn't need to modify any models, controllers or javascript.

System Requirements & Setup

This project requires Ruby 2.3.1. To install ruby we recommend using RVM (https://rvm.io/rvm/install). One you have RVM installed you can run: 

rvm install ruby-2.3.1

After ruby is installed, change into the project's root directory and install all dependencies:

bundle install

(If you had trouble with on that step, try this workaround*.)

After all the dependencies have been installed you should be able to start the server. Everything should already be setup and all you should need to do is run 

rails server

If you navigate to http://localhost:3000 you should see the page you will see our home page. From the home page you can navigate to our categories page http://localhost:3000/categories/1. This will be the page you are going to implement.

Finally since we have removed the root git repository from the app your final task should be to initialize one and make your first commit.

git init; git add .; git commit

At this point you should be good to go! If you run into any issues getting set up, please let us know ASAP.

Implementation: Home Page

We realize that most of the time when developing a project you will have a set of examples to pull and learn from that follow the guides. To that end we have included a version of our homepage that is styled, responsive, and sets an example for what you should be shooting for.

If you navigate to http://localhost:3000 you will be able to see it. The relevant styles for that page live app/assets/stylesheets/store/views/pages/home.scss and the html lives app/views/store/pages/home.html.erb. As you can see the CSS and the HTML have almost identical paths (this is on purpose).

If you open up the home.scss file you will immediately see components that are organized together with comment tags such as // -- Product Formats --. You should strive for a similar level of organization for your final css in your submission.

You will also notice, just as our FRONTEND.md file mentions, that we break our components into class tags, give semantic names with minimal parent selectors, no !important calls, and grouped media queries. We will be grading you on how well you can apply those guidelines to your own work as well.

Grading Criteria

  • Visual: Mocks and implementation look exactly alike
  • Responsive: Supports mobile and desktop beautifully
  • CrossBrowser support: Chrome, and Firefox (latest) -- Don't worry about IE
  • Style: How well you followed our style guide and reference implementation

Note: Timing will not be part of the grading criteria but will be required with your submission. If you are invited back for a team interview there will be a coding exercise and the time given will be proportional to what you reported for the assignment.

Assignment: The Category Page

Your task in this simulation is to add HTML & ERB to app/views/store/categories/show.html.erb and CSS to app/assets/stylesheets/store/views/categories/show.scss. (Again no JS should be used in the simulation) and very little ruby should be used (absolutely no models or controllers should be built , just view helpers like image_tag when needed). You can get an idea of what view helpers might be needed from the home.html.erb file.

We have already done the layout (header, footer, etc) for you so you are not responsible for those parts.

The mocks you should use as your UI guide can be found under mocks/. To give you an idea here is the desktop version:

You will need to do both desktop and mobile. Full size versions are in the mocks directory. You can fill in the text descriptions using Lorum Ipsum for speed

We have also included a couple example products and their images. You can find them in app/assets/images/store/products. Feel free to use those for the images. You can repeat them as necessary to file up the page with at least 12 products.

Since we are not allowing JS here none of the interaction here has to work, it just needs to look exactly like the mocks using HTML, CSS and Rails view helpers.

Troubleshooting installing nokogiri

Some environments won't run bundle install completely without a hitch the first time. If you get errors, try this:

brew uninstall xz
gem install nokogiri -v 1.6.8
bundle install

(Longer explaination of why that might work, and also nokogiri's docs on installation.)
